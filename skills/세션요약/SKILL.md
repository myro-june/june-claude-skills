현재 세션의 대화 내용을 한줄 브리핑으로 요약합니다.

## Workflow

### Step 1: 세션 컨텍스트 수집

현재 대화 컨텍스트에서 지금까지 나눈 내용을 분석한다.

### Step 2: 핵심 항목 추출

다음을 파악한다:
- **요청**: 사용자가 요청한 작업 목록
- **완료**: 성공적으로 끝난 작업
- **진행중**: 아직 완료되지 않은 작업
- **결정사항**: 주요 의사결정이나 발견된 이슈

### Step 3: 출력

#### 기본 모드 (`/세션요약`)

한줄 브리핑만 출력:

```
📋 세션 요약: {한줄 요약 문장}
```

#### 상세 모드 (`/세션요약 --detail`)

한줄 브리핑 + 항목별 정리:

```
📋 세션 요약: {한줄 요약 문장}

  ✅ 완료: {완료 항목 나열}
  🔄 진행중: {진행중 항목 나열}
  📌 결정사항: {주요 결정 나열}
```

## Rules

- 한줄 요약은 **반드시 한 문장**으로 작성한다.
- "무엇을 했고 어떻게 됐는지"에 초점을 맞춘다. 기술적 세부사항은 최소화.
- 진행 중인 작업이 있으면 현재 상태를 포함한다.
- 여러 주제를 다룬 세션의 경우, 가장 최근 또는 가장 핵심적인 작업을 중심으로 요약하고, 상세 모드에서 나머지를 나열한다.
- 대화가 거의 없는 경우: `📋 세션 요약: 세션 시작 직후, 아직 진행된 작업 없음.`
- `--detail` 플래그가 없으면 한줄 브리핑만 출력한다. 추가 설명 금지.
- 인식되지 않는 옵션이 전달된 경우, 기본 모드로 동작하고 지원되는 옵션(`--detail`)을 안내한다.
