# 슬랙 메시지 요약 스킬

슬랙 메시지 링크에서 원문 기사를 읽고 요약하여 해당 메시지의 스레드에 댓글로 작성합니다.

## When to Activate

사용자가 명시적으로 `/슬랙기사링크요약 <슬랙 메시지 링크>`를 입력했을 때만 실행.

## Workflow

1. **슬랙 메시지 링크 파싱**
   - URL 형식: `https://{workspace}.slack.com/archives/{channel_id}/p{timestamp}`
   - `channel_id` 추출
   - `p{timestamp}` → `message_ts` 변환: 뒤에서 6자리 앞에 `.` 삽입 (예: `p1774071352435959` → `1774071352.435959`)

2. **메시지 읽기**
   - `slack_read_thread`로 해당 메시지 내용 조회
   - 메시지 본문에서 외부 링크(URL) 추출

3. **원문 기사 가져오기**
   - `WebFetch`로 추출한 URL 접속하여 본문 전체 내용 수집
   - **반드시 별도의 `WebFetch` 요청으로 댓글/코멘트만 따로 수집** (첫 요청에서 댓글이 누락될 수 있음)
   - 두 번의 WebFetch를 병렬로 실행: 하나는 본문용, 하나는 댓글용

4. **요약 작성**
   - 기사 제목, 핵심 내용, 제안/결론을 요약
   - 댓글이 있으면 주요 반응도 요약에 포함

5. **슬랙 스레드에 게시**
   - `slack_send_message`로 해당 메시지의 스레드에 요약 댓글 작성
   - `channel_id`와 `thread_ts`(원본 메시지의 message_ts) 사용
   - 사용자 확인 없이 바로 게시

## 메시지 포맷 규칙

슬랙 메시지 가독성을 위해 반드시 아래 규칙을 따른다:

### 사용 금지 문자
- `ㅤ`(U+3164, 한글 필러) 절대 사용 금지 — Slack이 들여쓰기로 해석
- `*bold*`, `_italic_` 섹션 제목에 사용 금지 — Slack이 리스트 마커로 오인하여 들여쓰기 유발
- `---` 구분선 사용 금지 — Slack 미지원
- `~` (틸드) 사용 금지 — Slack이 `~text~`를 취소선으로 해석. 경로는 `$HOME/` 사용, 범위는 `-`로 표기 (예: 80-90%)

### 섹션 제목
- `[제목]` 형태로 대괄호로 감싸기
- 볼드/이탤릭 사용하지 않음

### 리스트 항목
- `•`로 표시
- **모든 리스트(불릿 `•`, 숫자 `1.` 등)는 반드시 마지막 섹션에만 사용** — Slack이 리스트 뒤의 콘텐츠를 리스트 내부로 흡수하여 다음 섹션 제목이 밀림(들여쓰기됨). 리스트 섹션 앞에는 반드시 텍스트 전용 섹션이 와야 함.
- 리스트가 아닌 섹션은 일반 텍스트 줄로 나열

### 섹션 간격
- `⠀`(U+2800, braille pattern blank)를 단독 줄로 넣어 시각적 구분
- `\n\n`(빈 줄)만으로는 Slack이 합쳐버리므로 반드시 `⠀` 사용

### 제목과 인트로
- 첫 줄: 기사 제목 (따옴표로 감싸고 "요약" 붙임)
- 바로 다음 줄(`\n`): 한줄 소개

포맷 예시:
```
"기사 제목" 요약
한줄 설명.
⠀
[문제점]
설명 텍스트 (불릿 없음)
⠀
[해결책]
항목 1 텍스트 (불릿 없음)
항목 2 텍스트
⠀
[댓글 반응]
• 반응 1
• 반응 2
```

## Examples

```
/슬랙기사링크요약 https://s-tat.slack.com/archives/C0AH3GV8803/p1774071352435959
```

## Notes

- 요약 완료 후 사용자 확인 없이 바로 스레드에 게시
- 원문 링크가 없는 메시지인 경우 메시지 자체 내용을 요약
- 댓글이 없으면 댓글 반응 섹션 생략
- 섹션이 2개 이하이고 리스트가 필요한 경우에도 리스트 섹션(불릿/숫자 모두)은 항상 마지막에 배치
